# 서울도시기본계획 PDF 관찰 메모

대상 문서: `2030 서울도시기본계획` 본보고서  
원본 URL: https://urban.seoul.go.kr/UpisArchive/DATA/PWEB/STATIC/1_seoul_plan.pdf

## 확인한 특징

- PDF는 225쪽, 약 51MB다.
- PyMuPDF 기준 텍스트 레이어는 존재하지만 많은 페이지에서 한글이 깨진다. 이 문서는 PDF 텍스트 추출만으로 청킹하면 검색 품질이 낮아질 가능성이 높다.
- 목차상 큰 구조는 6장과 부록으로 나뉜다. 특히 제3장 `핵심이슈별 계획`, 제4장 `공간구조 및 토지이용계획`, 제5장 `생활권계획`이 검색 가치가 높다.
- 페이지에는 본문 단락, 불릿, 표, 그래프, 지도, 범례, 그림 캡션, 출처/주석이 혼재한다.
- 표는 도시기본계획 변천처럼 연도, 계획명, 미래상, 계획기조를 비교하는 형태가 있다.
- 그래프는 인구구조, 1·2인 가구 증가 등 수치·연도·축 정보를 포함한다.
- 지도는 권역, 중심지, 발전축, 생활권, 거점, 하천/산지 같은 공간 관계를 담는다. 단순 OCR보다 VLM 요약과 별도 이미지 임베딩이 필요하다.
- 일부 장 전환/부록 페이지는 텍스트가 거의 없고 시각적 제목만 있다.
- 지도 페이지는 텍스트 양이 적어도 의미 밀도가 높으므로 `low_text_pages`를 버리면 안 된다.

## 청킹 시 주의점

- 목차 기반 장/절 범위를 1차 메타데이터로 부여해야 한다.
- 페이지 단위 청킹만으로는 길고 넓은 표·지도 맥락을 잃을 수 있다. 페이지 chunk와 figure/table/map chunk를 동시에 만들어야 한다.
- 표/그래프는 본문 단락과 다른 schema로 저장해야 한다. `표 제목`, `행/열`, `수치`, `출처`, `설명`을 분리한다.
- 지도는 `지도 제목`, `공간 객체`, `관계`, `정책 방향`, `범례`를 함께 저장한다.
- PDF 텍스트가 깨지는 페이지는 OCR 결과를 우선하고, OCR도 불충분하면 VLM page summary를 검색 텍스트에 포함한다.
- dense embedding만 쓰면 행정용어, 지명, 정책명 정확도가 떨어질 수 있으므로 BM25 lexical index를 같이 만든다.
- 질의 시 dense, BM25, graph expansion을 결합하고 RRF 같은 late fusion으로 랭킹을 합친다.
